🔄 Four Rotation Types

LL Rotation (Right Rotate)

Left-Left case: Node inserted in left subtree of left child

z y / \ / \ y T4 ──────▶ x z / \ / \ / \ x T3 T1 T2 T3 T4

RR Rotation (Left Rotate)

Right-Right case: Node inserted in right subtree of right child

z y / \ / \ T1 y ──────▶ z x / \ / \ / \ T2 x T1 T2 T3 T4

LR Rotation (Left-Right)

Left-Right case: Node inserted in right subtree of left child

z z x / \ / \ / \ y T4 ──▶ x T4 ──▶ y z / \ / \ / \ / \ T1 x y T3 T1 T2 T3 T4 / \ / \ T2 T3 T1 T2

RL Rotation (Right-Left)

Right-Left case: Node inserted in left subtree of right child

z z x / \ / \ / \ T1 y ──▶ T1 x ──▶ z y / \ / \ / \ / \ x T4 T2 y T1 T2 T3 T4 / \ / \ T2 T3 T3 T4

💡 AVL Property

Named after inventors Adelson-Velsky and Landis (1962).

Balance Factor

BF(node) = height(left) - height(right)

For every node:

  • BF = -1, 0, +1 → Balanced
  • BF = +2 → Left-heavy (needs rotation)
  • BF = -2 → Right-heavy (needs rotation)
✅ Guarantee: Height is always O(log n), ensuring all operations are O(log n).

⏱️ Complexity Analysis

Operation Time
Search O(log n)
Insert O(log n)
Delete O(log n)
Rotation O(1)
Space O(n)

AVL vs BST

BST Worst AVL
Search O(n) O(log n)
Insert O(n) O(log n)

🔄 When to Rotate

After insertion, check balance factor going up to root:

BF = +2 (Left-heavy)

  • If left child BF ≥ 0 → LL (Right Rotate)
  • If left child BF < 0 → LR (Left-Right)

BF = -2 (Right-heavy)

  • If right child BF ≤ 0 → RR (Left Rotate)
  • If right child BF > 0 → RL (Right-Left)

🤔 When to Use AVL

✅ Good For:

  • Lookup-intensive applications
  • Databases (indexing)
  • When worst-case matters

❌ Consider Red-Black Trees When:

  • Frequent insertions/deletions
  • Slightly relaxed balance OK
  • (Red-Black trees do fewer rotations)
