💡 BST Property

For every node in a Binary Search Tree:

  • All values in the left subtree are smaller
  • All values in the right subtree are larger
       50
      /  \
    30    70
   /  \   /  \
  20  40 60  80

In-order traversal gives: 20, 30, 40, 50, 60, 70, 80 (sorted!)

⏱️ Complexity Analysis

Operation Average Worst
Search O(log n) O(n)
Insert O(log n) O(n)
Delete O(log n) O(n)
Space O(n)
⚠️ Worst Case: Inserting sorted data creates a "linked list" shape, degrading to O(n). Use balanced trees (AVL, Red-Black) to guarantee O(log n).

🔧 Operations

Insert

  1. Start at root
  2. If value < node, go left
  3. If value > node, go right
  4. Insert when reaching null

Search

  1. Start at root
  2. Compare with current node
  3. Go left/right based on comparison
  4. Return when found or null

Delete

Three cases:

  • Leaf: Simply remove
  • One child: Replace with child
  • Two children: Replace with in-order successor
