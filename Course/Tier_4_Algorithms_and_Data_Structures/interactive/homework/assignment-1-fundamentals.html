Problem 1 15 points

Determine the time complexity of each code snippet:

a) (5 pts)

def func_a(n):
    total = 0
    for i in range(n):
        for j in range(n):
            total += i * j
    return total

b) (5 pts)

def func_b(n):
    total = 0
    for i in range(n):
        for j in range(i):
            total += 1
    return total

c) (5 pts)

def func_c(n):
    i = n
    while i > 1:
        print(i)
        i = i // 2

Answer Key

a) O(n²) - Two nested loops, each running n times. Total: n × n = n²

b) O(n²) - Inner loop runs 0 + 1 + 2 + ... + (n-1) = n(n-1)/2 = O(n²)

c) O(log n) - i is halved each iteration. Loop runs log₂(n) times.

Problem 3 20 points

Analyze this recursive function:

def mystery(n):
    if n <= 1:
        return 1
    return mystery(n - 1) + mystery(n - 1)

a) (5 pts) What does this function compute?

b) (10 pts) What is the time complexity? Explain using a recurrence relation.

c) (5 pts) What is the space complexity?

Answer Key

a) The function computes 2^(n-1) or equivalently 2ⁿ⁻¹ (powers of 2)

b) Time Complexity: O(2ⁿ)

Recurrence relation: T(n) = 2T(n-1) + O(1)

T(n) = 2T(n-1) + 1
     = 2(2T(n-2) + 1) + 1
     = 4T(n-2) + 2 + 1
     = 8T(n-3) + 4 + 2 + 1
     = 2ᵏ T(n-k) + 2ᵏ⁻¹ + ... + 1

When k = n-1: T(n) = 2ⁿ⁻¹ T(1) + 2ⁿ⁻² + ... + 1
            = 2ⁿ⁻¹ + (2ⁿ⁻¹ - 1)
            = 2ⁿ - 1 = O(2ⁿ)

c) Space Complexity: O(n)

Maximum depth of recursion stack is n (one branch goes all the way down before backtracking).

Problem 4 15 points

For each scenario, determine the most appropriate time complexity bound:

a) (5 pts)

Finding the minimum element in an unsorted array of n elements.

b) (5 pts)

Checking if a sorted array contains a specific element.

c) (5 pts)

Finding all pairs of elements in an array that sum to a target value.

Answer Key

a) O(n) - Must examine every element once. Cannot do better than O(n) for unsorted array.

b) O(log n) - Binary search can find element or determine absence in O(log n) time.

c) O(n²) naive, O(n log n) with sorting, O(n) with hash table

  • Naive: Check all pairs - O(n²)
  • Sort + two pointers: O(n log n)
  • Hash set: For each element, check if complement exists - O(n)

Problem 5 15 points

True or False (explain your reasoning):

a) (5 pts)

If f(n) = O(n²), then f(n) = O(n³)

b) (5 pts)

If f(n) = O(n²) and g(n) = O(n), then f(n) + g(n) = O(n²)

c) (5 pts)

If f(n) = O(log n), then f(n) = O(n)

Answer Key

a) TRUE - Big O is an upper bound. If f(n) ≤ cn² for large n, then f(n) ≤ cn³ as well (since n² ≤ n³ for n ≥ 1).

b) TRUE - The sum is dominated by the larger term. n² + n = O(n²).

c) TRUE - log n grows slower than n, so log n ≤ n for sufficiently large n. Therefore O(log n) ⊂ O(n).

Problem 6 20 points

Analyze the following algorithm:

def process(arr):
    n = len(arr)
    result = []
    
    for i in range(n):                    # Loop A
        for j in range(n):                # Loop B
            result.append(arr[i] + arr[j])
    
    sorted_result = sorted(result)        # Sort
    
    unique = []
    for item in sorted_result:            # Loop C
        if not unique or unique[-1] != item:
            unique.append(item)
    
    return unique

a) (5 pts) What is the time complexity of Loop A + Loop B?

b) (5 pts) What is the size of result after the nested loops?

c) (5 pts) What is the time complexity of the Sort?

d) (5 pts) What is the overall time complexity?

Answer Key

a) O(n²) - Two nested loops, each running n times.

b) n² elements - The nested loops add n × n = n² elements to result.

c) O(n² log n²) = O(n² log n) - Sorting n² elements. Note: log(n²) = 2 log n = O(log n).

d) O(n² log n)

  • Nested loops: O(n²)
  • Sorting: O(n² log n)
  • Deduplication: O(n²)
  • Total: O(n²) + O(n² log n) + O(n²) = O(n² log n)

The sorting step dominates.

Bonus Problem 10 points (extra credit)

Prove or disprove: For any two functions f(n) and g(n), either f(n) = O(g(n)) or g(n) = O(f(n)).

Answer Key

DISPROVE - This statement is false.

Counterexample:

f(n) = n      (when n is even)
f(n) = 1      (when n is odd)

g(n) = 1      (when n is even)
g(n) = n      (when n is odd)

Neither function dominates the other for all large n:

  • For even n: f(n) = n > 1 = g(n), so g ≠ O(f)
  • For odd n: g(n) = n > 1 = f(n), so f ≠ O(g)

Simpler example: f(n) = n for even n, 1 otherwise; g(n) = n for odd n, 1 otherwise.
