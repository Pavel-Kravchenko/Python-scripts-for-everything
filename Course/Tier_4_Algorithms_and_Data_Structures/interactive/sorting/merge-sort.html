💡 How It Works

1. Divide

Split the array in half recursively until each subarray has one element.

2. Conquer

Single-element arrays are inherently sorted.

3. Combine (Merge)

Merge sorted subarrays by comparing elements and placing them in order.

🎯 Key Insight: The merge step is where the actual sorting happens. Two sorted arrays can be merged in O(n) time.

⏱️ Complexity Analysis

Case Time
Best O(n log n)
Average O(n log n)
Worst O(n log n)
Space O(n)

Properties

  • ✅ Stable: Equal elements maintain order
  • ✅ Predictable: Always O(n log n)
  • ✅ Parallelizable: Subarrays can be sorted independently
  • ❌ Not in-place: Requires O(n) extra space

🤔 When to Use

✅ Good For:

  • Linked lists (natural fit, O(1) space)
  • External sorting (large files)
  • Stable sort requirement
  • Predictable performance needed
  • Parallel processing

❌ Avoid When:

  • Memory is very limited
  • Small arrays (overhead not worth it)
  • In-place sorting required
