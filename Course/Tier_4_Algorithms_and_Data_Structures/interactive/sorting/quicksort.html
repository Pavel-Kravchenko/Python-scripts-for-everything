📝 Execution Steps

💡 How It Works

1. Choose a Pivot

Select an element as the pivot (commonly last, first, or random).

2. Partition

Rearrange elements so that:

  • Elements < pivot go to the left
  • Elements ≥ pivot go to the right
  • Pivot ends up in its final sorted position

3. Recurse

Recursively apply to left and right subarrays.

⚠️ Pivot Choice Matters! Poor pivot selection (e.g., always choosing min/max) leads to O(n²) worst case.

⏱️ Complexity Analysis

Case Time
Best O(n log n)
Average O(n log n)
Worst O(n²)
Space O(log n)

Properties

  • ✅ In-place: O(log n) stack space
  • ✅ Cache-friendly: Sequential memory access
  • ✅ Fast in practice: Low constant factors
  • ❌ Not stable: May reorder equal elements
  • ❌ Worst case: O(n²) with bad pivots

1def quicksort(arr, low, high):
2    if low < high:
3        pi = partition(arr, low, high)
4        quicksort(arr, low, pi - 1)
5        quicksort(arr, pi + 1, high)
6
7def partition(arr, low, high):
8    pivot = arr[high]  # Last element as pivot
9    i = low - 1  # Index of smaller element
10    
11    for j in range(low, high):
12        if arr[j] < pivot:
13            i += 1
14            arr[i], arr[j] = arr[j], arr[i]
15    
16    arr[i + 1], arr[high] = arr[high], arr[i + 1]
17    return i + 1

🤔 When to Use

✅ Good For:

  • General-purpose sorting
  • Arrays (cache efficiency)
  • When average case matters most
  • Memory-constrained environments

❌ Avoid When:

  • Stability is required
  • Worst-case guarantee needed
  • Nearly sorted data (use Insertion Sort)
  • Linked lists (use Merge Sort)
