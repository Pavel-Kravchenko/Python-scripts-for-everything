Exercise 1 Easy

Linked List Reversal

Given the following singly linked list, show the state after reversing it:

1 → 2 → 3 → 4 → 5 → NULL

What are the values of prev, current, and next after each iteration of the reversal algorithm?

Answer:

Final reversed list:

5 → 4 → 3 → 2 → 1 → NULL

Step-by-step:

Initial: prev=NULL, current=1, next=2 Step 1: prev=1, current=2, next=3 (1→NULL) Step 2: prev=2, current=3, next=4 (2→1→NULL) Step 3: prev=3, current=4, next=5 (3→2→1→NULL) Step 4: prev=4, current=5, next=NULL (4→3→2→1→NULL) Step 5: prev=5, current=NULL (5→4→3→2→1→NULL)
Exercise 2 Easy

Stack Operations

Given an empty stack, perform the following operations and show the stack state after each:

push(5), push(3), push(8), pop(), push(2), pop(), pop()

What is the final value on top of the stack?

Answer:

push(5): [5] (bottom → top) push(3): [5, 3] push(8): [5, 3, 8] pop(): [5, 3] returns 8 push(2): [5, 3, 2] pop(): [5, 3] returns 2 pop(): [5] returns 3

Final top value: 5

Exercise 3 Medium

Balanced Parentheses

Using a stack, determine if the following expressions have balanced brackets:

  1. ((()))
  2. ([{}])
  3. ((])
  4. {[()]}
  5. [(])

Answer:

  1. ((())) → ✓ Balanced
  2. ([{}]) → ✓ Balanced
  3. ((]) → ✗ Unbalanced (] doesn't match ()
  4. {[()]} → ✓ Balanced
  5. [(]) → ✗ Unbalanced (interleaved, not nested)

Algorithm: Push opening brackets, pop on closing brackets and check match.

Exercise 4 Medium

Queue vs Two-Stack Queue

Implement a queue using two stacks. Show the state of both stacks after:

enqueue(1), enqueue(2), enqueue(3), dequeue(), enqueue(4), dequeue()

Answer:

enqueue(1): stack_in=[1], stack_out=[] enqueue(2): stack_in=[1,2], stack_out=[] enqueue(3): stack_in=[1,2,3], stack_out=[] dequeue(): Transfer: stack_in=[], stack_out=[3,2,1] Pop: stack_out=[3,2] → returns 1 enqueue(4): stack_in=[4], stack_out=[3,2] dequeue(): stack_out not empty, just pop stack_out=[3] → returns 2 Final: stack_in=[4], stack_out=[3]
Exercise 5 Medium

Floyd's Cycle Detection

A linked list has the following structure with a cycle:

1 → 2 → 3 → 4 → 5 ↑ ↓ └───────┘ (5 points back to 3)

Trace Floyd's algorithm (slow/fast pointers). Where do they meet?

Answer:

Start: slow=1, fast=1 Step 1: slow=2, fast=3 Step 2: slow=3, fast=5 Step 3: slow=4, fast=4 ← MEET!

They meet at node 4.

Fast moves 2 steps per iteration, slow moves 1. If there's a cycle, they must meet.

Exercise 6 Medium

Hash Table Insertion with Chaining

Insert the following keys into a hash table of size 7 using h(k) = k mod 7 with separate chaining:

Keys: 15, 22, 8, 29, 36, 1, 43

Show the final state of the hash table.

Answer:

15 mod 7 = 1 22 mod 7 = 1 (collision with 15) 8 mod 7 = 1 (collision with 15, 22) 29 mod 7 = 1 (collision!) 36 mod 7 = 1 (collision!) 1 mod 7 = 1 (collision!) 43 mod 7 = 1 (collision!) Final Hash Table: [0]: empty [1]: 15 → 22 → 8 → 29 → 36 → 1 → 43 [2]: empty [3]: empty [4]: empty [5]: empty [6]: empty

All keys hash to index 1! This is a degenerate case showing why good hash functions matter.

Exercise 7 Hard

Linear Probing with Deletion

Given a hash table of size 7 with linear probing, insert then delete:

Insert: 10, 17, 24, 31 (all hash to 3) Delete: 17 Search: 31

Why do we need tombstones (deletion markers)?

Answer:

After inserts: [0]: empty [1]: empty [2]: empty [3]: 10 [4]: 17 [5]: 24 [6]: 31 After delete 17 (with tombstone): [3]: 10 [4]: DELETED ← tombstone [5]: 24 [6]: 31 Search for 31: - h(31) = 3, found 10 ≠ 31, probe - Index 4: DELETED (continue probing!) - Index 5: 24 ≠ 31, probe - Index 6: 31 ✓ Found!

Why tombstones? Without them, the search for 31 would stop at the empty slot where 17 was, incorrectly concluding 31 doesn't exist.

Exercise 8 Hard

LRU Cache Design

Design an LRU (Least Recently Used) cache with O(1) get and put operations.

Hint: Combine two data structures we've learned.

What operations does each data structure handle?

Answer:

Use: Hash Table + Doubly Linked List

Hash Table: key → node pointer (O(1) lookup) Doubly Linked List: maintains order (O(1) move/remove) Structure: HEAD ↔ [MRU] ↔ ... ↔ [LRU] ↔ TAIL get(key): 1. Look up in hash table → O(1) 2. Move node to head (MRU) → O(1) 3. Return value put(key, value): 1. If exists: update value, move to head 2. If new: - Create node, add to head - Add to hash table - If over capacity: remove tail (LRU)

Both operations are O(1) because hash table provides direct access and doubly linked list allows O(1) removal/insertion.
