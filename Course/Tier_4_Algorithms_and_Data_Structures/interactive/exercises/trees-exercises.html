Exercise 1: BST Property Verification

Which of these trees are valid Binary Search Trees?

Tree A:        Tree B:        Tree C:
    5              5              5
   / \            / \            / \
  3   7          3   7          3   8
 / \   \        / \   \        / \   \
1   4   8      1   6   8      1   4   7

✅ Solution

  • Tree A: ✅ Valid BST
    Left subtree: 1, 3, 4 (all < 5)
    Right subtree: 7, 8 (all > 5)
    Each subtree also satisfies BST property
  • Tree B: ❌ Invalid BST
    6 is in the left subtree of 5, but 6 > 5
    The BST property requires ALL nodes in left subtree < root
  • Tree C: ❌ Invalid BST
    7 is in the right subtree of 8, but 7 < 8
    7 should be the left child of 8, not right child

Common mistake: Only checking immediate children. You must verify the property for the entire subtree!

Exercise 2: Tree Traversals

Given this tree:

        1
       / \
      2   3
     / \   \
    4   5   6

Write out the nodes visited for each traversal:

  1. Pre-order
  2. In-order
  3. Post-order
  4. Level-order (BFS)

✅ Solution

Traversal Order Rule Result
Pre-order Root → Left → Right 1, 2, 4, 5, 3, 6
In-order Left → Root → Right 4, 2, 5, 1, 3, 6
Post-order Left → Right → Root 4, 5, 2, 6, 3, 1
Level-order Level by level 1, 2, 3, 4, 5, 6

Memory trick:

  • Pre = root PRE-cedes children
  • In = root is IN between children
  • Post = root POST-cedes (comes after) children

Exercise 3: BST Insertion Order

Given the insertion order [50, 30, 70, 20, 40, 60, 80], draw the resulting BST.

Then answer: What is the height of the tree?

✅ Solution

        50
       /  \
      30   70
     / \   / \
    20 40 60 80

Height = 3 (or 2 if counting edges)

Step by step:

  1. Insert 50 → root
  2. Insert 30 → 30 < 50, left child of 50
  3. Insert 70 → 70 > 50, right child of 50
  4. Insert 20 → 20 < 50, 20 < 30, left child of 30
  5. Insert 40 → 40 < 50, 40 > 30, right child of 30
  6. Insert 60 → 60 > 50, 60 < 70, left child of 70
  7. Insert 80 → 80 > 50, 80 > 70, right child of 70

Note: This is a perfectly balanced BST! Different insertion orders can produce very different tree shapes.

Exercise 4: AVL Balance Factors

Calculate the balance factor for each node:

        30
       /  \
      20   40
     /       \
    10       50
   /
  5

Is this a valid AVL tree? If not, what rotation(s) are needed?

✅ Solution

Balance Factor = height(left) - height(right)

Node 5:  BF = 0 - 0 = 0   ✓
Node 10: BF = 1 - 0 = 1   ✓
Node 20: BF = 2 - 0 = 2   ✗ IMBALANCED!
Node 50: BF = 0 - 0 = 0   ✓
Node 40: BF = 0 - 1 = -1  ✓
Node 30: BF = 3 - 2 = 1   ✓

        30 (BF=1)
       /  \
      20   40 (BF=-1)
(BF=2)/     \
    10      50 (BF=0)
   /
  5

This is NOT a valid AVL tree because node 20 has BF = 2.

Fix: LL Case at node 20 → Right Rotation

After right rotation at 20:

        30
       /  \
      10   40
     /  \    \
    5   20   50

All nodes now have |BF| ≤ 1 ✓

Exercise 5: BST Deletion Cases

Starting with this BST:

        50
       /  \
      30   70
     / \   / \
    20 40 60 80

Show the tree after each deletion:

  1. Delete 20 (leaf)
  2. Delete 30 (one child)
  3. Delete 50 (two children)

✅ Solution

1. Delete 20 (Leaf Node)
        50                    50
       /  \                  /  \
      30   70      →       30   70
     / \   / \            /  \  / \
    20 40 60 80          - 40 60 80

Simply remove the leaf.
2. Delete 30 (One Child - 40)
        50                    50
       /  \                  /  \
      30   70      →       40   70
        \  / \                 / \
       40 60 80              60 80

Replace node with its only child.
3. Delete 50 (Two Children)
        50                    60
       /  \                  /  \
      40   70      →       40   70
          / \                    \
         60 80                   80

Replace with in-order successor (60),
then delete 60 from its original position.

In-order successor: The smallest node in the right subtree (leftmost node of right subtree).

Exercise 6: Validate BST

Write code to check if a binary tree is a valid BST.

class Node:
    def __init__(self, val):
        self.val = val
        self.left = None
        self.right = None

def is_valid_bst(root):
    # Your code here
    pass

✅ Solution

# Approach 1: Min/Max Range
def is_valid_bst(root, min_val=float('-inf'), max_val=float('inf')):
    if not root:
        return True
    
    if root.val <= min_val or root.val >= max_val:
        return False
    
    return (is_valid_bst(root.left, min_val, root.val) and
            is_valid_bst(root.right, root.val, max_val))

# Approach 2: In-order Traversal
def is_valid_bst_inorder(root):
    prev = [float('-inf')]  # Use list to maintain state
    
    def inorder(node):
        if not node:
            return True
        
        if not inorder(node.left):
            return False
        
        if node.val <= prev[0]:
            return False
        prev[0] = node.val
        
        return inorder(node.right)
    
    return inorder(root)

# Why does in-order work?
# In-order traversal of a BST produces sorted output.
# If at any point current <= previous, it's not a valid BST.

Exercise 7: Lowest Common Ancestor

Find the lowest common ancestor (LCA) of two nodes in a BST.

        6
       / \
      2   8
     / \ / \
    0  4 7  9
      / \
     3   5

LCA(2, 8) = 6
LCA(2, 4) = 2
LCA(3, 5) = 4

✅ Solution

def lowest_common_ancestor(root, p, q):
    # In a BST, we can use the BST property!
    while root:
        if p.val < root.val and q.val < root.val:
            # Both nodes are in left subtree
            root = root.left
        elif p.val > root.val and q.val > root.val:
            # Both nodes are in right subtree
            root = root.right
        else:
            # Split point: one node on each side, or we found one of them
            return root
    return None

# Time: O(h) where h is height
# Space: O(1) iterative, O(h) recursive

# For a general binary tree (not BST):
def lca_binary_tree(root, p, q):
    if not root or root == p or root == q:
        return root
    
    left = lca_binary_tree(root.left, p, q)
    right = lca_binary_tree(root.right, p, q)
    
    if left and right:
        return root  # p and q are on different sides
    return left if left else right

Exercise 8: Tree Height and Balance

Write a function that checks if a binary tree is height-balanced (AVL property: for every node, height difference of subtrees ≤ 1).

✅ Solution

# Naive approach: O(n²) - compute height at each node
def is_balanced_naive(root):
    if not root:
        return True
    
    left_height = get_height(root.left)
    right_height = get_height(root.right)
    
    if abs(left_height - right_height) > 1:
        return False
    
    return is_balanced_naive(root.left) and is_balanced_naive(root.right)

def get_height(node):
    if not node:
        return 0
    return 1 + max(get_height(node.left), get_height(node.right))


# Optimal approach: O(n) - check balance while computing height
def is_balanced(root):
    def check(node):
        """Returns (is_balanced, height)"""
        if not node:
            return True, 0
        
        left_balanced, left_height = check(node.left)
        if not left_balanced:
            return False, 0
        
        right_balanced, right_height = check(node.right)
        if not right_balanced:
            return False, 0
        
        balanced = abs(left_height - right_height) <= 1
        height = 1 + max(left_height, right_height)
        
        return balanced, height
    
    return check(root)[0]

# Alternative: Return -1 for unbalanced
def is_balanced_v2(root):
    def height(node):
        if not node:
            return 0
        
        left = height(node.left)
        if left == -1:
            return -1
        
        right = height(node.right)
        if right == -1:
            return -1
        
        if abs(left - right) > 1:
            return -1
        
        return 1 + max(left, right)
    
    return height(root) != -1
