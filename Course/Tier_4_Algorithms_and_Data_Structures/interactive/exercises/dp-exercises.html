📋 Dynamic Programming Exercises

8 classic DP problems with solutions

1. Climbing Stairs Easy

You are climbing a staircase with n steps. Each time you can climb 1 or 2 steps. How many distinct ways can you reach the top?

Example:
Input: n = 5
Output: 8
Ways: 1+1+1+1+1, 1+1+1+2, 1+1+2+1, 1+2+1+1, 2+1+1+1, 
      1+2+2, 2+1+2, 2+2+1
To reach step n, you either came from step n-1 (one step) or step n-2 (two steps). This is exactly like Fibonacci!
dp[i] = dp[i-1] + dp[i-2]
# Bottom-up with O(1) space
def climbStairs(n):
    if n <= 2:
        return n
    
    prev2, prev1 = 1, 2
    for i in range(3, n + 1):
        curr = prev1 + prev2
        prev2, prev1 = prev1, curr
    
    return prev1

# Memoization version
def climbStairsMemo(n, memo={}):
    if n <= 2:
        return n
    if n in memo:
        return memo[n]
    
    memo[n] = climbStairsMemo(n-1, memo) + climbStairsMemo(n-2, memo)
    return memo[n]
Time: O(n)
Space: O(1) optimized, O(n) with memoization

2. House Robber Medium

You are a robber planning to rob houses along a street. Each house has money, but adjacent houses have security systems that will alert police if both are robbed. Find the maximum amount you can rob.

Example:
Input: nums = [2, 7, 9, 3, 1]
Output: 12
Explanation: Rob houses 0, 2, and 4 → 2 + 9 + 1 = 12
For each house i, you have two choices: rob it (add nums[i] to max from i-2) or skip it (take max from i-1). Track both possibilities.
dp[i] = max(dp[i-1], dp[i-2] + nums[i])
def rob(nums):
    if not nums:
        return 0
    if len(nums) == 1:
        return nums[0]
    
    # prev2 = max money up to 2 houses ago
    # prev1 = max money up to previous house
    prev2, prev1 = 0, 0
    
    for num in nums:
        curr = max(prev1, prev2 + num)
        prev2, prev1 = prev1, curr
    
    return prev1

# Example trace for [2, 7, 9, 3, 1]:
# i=0: curr = max(0, 0+2) = 2,   prev2=0, prev1=2
# i=1: curr = max(2, 0+7) = 7,   prev2=2, prev1=7
# i=2: curr = max(7, 2+9) = 11,  prev2=7, prev1=11
# i=3: curr = max(11, 7+3) = 11, prev2=11, prev1=11
# i=4: curr = max(11, 11+1) = 12
Time: O(n)
Space: O(1)

3. Coin Change Medium

Given coins of different denominations and a total amount, find the minimum number of coins needed to make up that amount. Return -1 if impossible.

Example:
Input: coins = [1, 2, 5], amount = 11
Output: 3
Explanation: 11 = 5 + 5 + 1
Build up from amount 0 to target. For each amount, try each coin and take the minimum. dp[amount] = min coins to make that amount.
dp[i] = min(dp[i], dp[i - coin] + 1) for each coin
def coinChange(coins, amount):
    # dp[i] = minimum coins to make amount i
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0  # Base case: 0 coins to make amount 0
    
    for i in range(1, amount + 1):
        for coin in coins:
            if coin <= i and dp[i - coin] != float('inf'):
                dp[i] = min(dp[i], dp[i - coin] + 1)
    
    return dp[amount] if dp[amount] != float('inf') else -1

# Example trace for coins=[1,2,5], amount=11:
# dp[0]=0, dp[1]=1, dp[2]=1, dp[3]=2, dp[4]=2
# dp[5]=1, dp[6]=2, dp[7]=2, dp[8]=3, dp[9]=3
# dp[10]=2, dp[11]=3
Time: O(amount × coins)
Space: O(amount)

4. Longest Increasing Subsequence Medium

Given an integer array, return the length of the longest strictly increasing subsequence.

Example:
Input: nums = [10, 9, 2, 5, 3, 7, 101, 18]
Output: 4
Explanation: [2, 3, 7, 101] or [2, 5, 7, 101]
dp[i] = length of LIS ending at index i. For each i, look at all j < i where nums[j] < nums[i]. O(n²) solution. For O(n log n), use binary search with a "tails" array.
dp[i] = max(dp[j] + 1) for all j < i where nums[j] < nums[i]
# O(n²) DP solution
def lengthOfLIS(nums):
    if not nums:
        return 0
    
    n = len(nums)
    dp = [1] * n  # Each element is an LIS of length 1
    
    for i in range(1, n):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    
    return max(dp)

# O(n log n) Binary Search solution
import bisect

def lengthOfLIS_optimized(nums):
    # tails[i] = smallest ending element of LIS with length i+1
    tails = []
    
    for num in nums:
        pos = bisect.bisect_left(tails, num)
        if pos == len(tails):
            tails.append(num)
        else:
            tails[pos] = num
    
    return len(tails)
Time: O(n²) or O(n log n) optimized
Space: O(n)

5. 0/1 Knapsack Medium

Given n items with weights and values, and a knapsack capacity W, find the maximum value you can fit. Each item can only be used once.

Example:
Input: weights = [1, 3, 4, 5], values = [1, 4, 5, 7], W = 7
Output: 9
Explanation: Take items with weight 3 and 4 → values 4 + 5 = 9
2D DP: dp[i][w] = max value using first i items with capacity w. For each item, either skip it or take it (if it fits).
dp[i][w] = max(dp[i-1][w], dp[i-1][w-weight[i]] + value[i])
# 2D DP solution
def knapsack(weights, values, W):
    n = len(weights)
    dp = [[0] * (W + 1) for _ in range(n + 1)]
    
    for i in range(1, n + 1):
        for w in range(W + 1):
            # Don't take item i
            dp[i][w] = dp[i-1][w]
            
            # Take item i (if it fits)
            if weights[i-1] <= w:
                dp[i][w] = max(dp[i][w], 
                    dp[i-1][w - weights[i-1]] + values[i-1])
    
    return dp[n][W]

# Space-optimized 1D solution
def knapsack_optimized(weights, values, W):
    dp = [0] * (W + 1)
    
    for i in range(len(weights)):
        # Iterate backwards to avoid using same item twice
        for w in range(W, weights[i] - 1, -1):
            dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
    
    return dp[W]
Time: O(n × W)
Space: O(W) optimized

6. Word Break Medium

Given a string s and a dictionary of words, determine if s can be segmented into a space-separated sequence of dictionary words.

Example:
Input: s = "leetcode", wordDict = ["leet", "code"]
Output: True
Explanation: "leetcode" = "leet" + "code"
dp[i] = True if s[0:i] can be segmented. For each position i, check all possible last words: if s[j:i] is in dict and dp[j] is True, then dp[i] is True.
dp[i] = any(dp[j] and s[j:i] in wordDict) for all j < i
def wordBreak(s, wordDict):
    word_set = set(wordDict)
    n = len(s)
    
    # dp[i] = True if s[0:i] can be segmented
    dp = [False] * (n + 1)
    dp[0] = True  # Empty string can be segmented
    
    for i in range(1, n + 1):
        for j in range(i):
            if dp[j] and s[j:i] in word_set:
                dp[i] = True
                break
    
    return dp[n]

# Example trace for "leetcode":
# i=1: "l" not in dict, dp[1]=False
# i=2: "le" not in dict, dp[2]=False
# i=3: "lee" not in dict, dp[3]=False
# i=4: "leet" in dict and dp[0]=True, dp[4]=True
# i=5-7: no valid word
# i=8: "code" in dict and dp[4]=True, dp[8]=True
Time: O(n² × k) where k = avg word length for substring comparison
Space: O(n + m) where m = dict size

7. Unique Paths Medium

A robot is at top-left corner of an m×n grid. It can only move right or down. How many unique paths exist to reach the bottom-right corner?

Example:
Input: m = 3, n = 7
Output: 28
To reach cell (i,j), you came from (i-1,j) or (i,j-1). Sum paths from both. First row and column have only 1 path each.
dp[i][j] = dp[i-1][j] + dp[i][j-1]
# 2D DP
def uniquePaths(m, n):
    dp = [[1] * n for _ in range(m)]
    
    for i in range(1, m):
        for j in range(1, n):
            dp[i][j] = dp[i-1][j] + dp[i][j-1]
    
    return dp[m-1][n-1]

# 1D space optimization
def uniquePaths_optimized(m, n):
    dp = [1] * n
    
    for i in range(1, m):
        for j in range(1, n):
            dp[j] += dp[j-1]
    
    return dp[n-1]

# Math solution: C(m+n-2, m-1)
from math import comb
def uniquePaths_math(m, n):
    return comb(m + n - 2, m - 1)
Time: O(m × n)
Space: O(n) optimized

8. Partition Equal Subset Sum Medium

Given an array of positive integers, determine if it can be partitioned into two subsets with equal sum.

Example:
Input: nums = [1, 5, 11, 5]
Output: True
Explanation: [1, 5, 5] and [11] both sum to 11
If total sum is odd, impossible. Otherwise, find if there's a subset with sum = total/2. This is a 0/1 knapsack variant!
dp[i] = True if sum i is achievable
def canPartition(nums):
    total = sum(nums)
    
    # If odd, can't partition equally
    if total % 2 != 0:
        return False
    
    target = total // 2
    
    # dp[i] = True if we can make sum i
    dp = [False] * (target + 1)
    dp[0] = True
    
    for num in nums:
        # Iterate backwards to avoid using same number twice
        for i in range(target, num - 1, -1):
            dp[i] = dp[i] or dp[i - num]
    
    return dp[target]

# Using set (alternative)
def canPartition_set(nums):
    total = sum(nums)
    if total % 2 != 0:
        return False
    
    target = total // 2
    achievable = {0}
    
    for num in nums:
        achievable = achievable | {x + num for x in achievable}
        if target in achievable:
            return True
    
    return target in achievable
Time: O(n × sum)
Space: O(sum)
