💡 How It Works

  1. Require sorted array
  2. Set left = 0, right = n-1
  3. Calculate mid = (left + right) / 2
  4. Compare target with arr[mid]:
    • If equal → found!
    • If target < mid → search left half
    • If target > mid → search right half
  5. Repeat until found or range is empty
🚀 Power of Halving: For 1 billion elements, binary search needs at most 30 comparisons!

💻 Python Code

binary_search.py Python 
1def binary_search(arr, target):
2    left, right = 0, len(arr) - 1
3    
4    while left <= right:
5        mid = (left + right) // 2
6        
7        if arr[mid] == target:
8            return mid  # Found!
9        elif arr[mid] < target:
10            left = mid + 1  # Search right
11        else:
12            right = mid - 1  # Search left
13    
14    return -1  # Not found

🔄 Common Variations

  • Lower Bound: Find first element ≥ target
  • Upper Bound: Find first element > target
  • Rotated Array: Search in rotated sorted array
  • 2D Search: Search in row/column sorted matrix

🤔 When to Use

✅ Requirements:

  • Array must be sorted
  • Random access (arrays, not linked lists)

✅ Good For:

  • Large sorted datasets
  • Repeated searches on same data
  • Finding boundaries/ranges

❌ Use Linear Search When:

  • Data is unsorted
  • Array is very small (n < 10)
  • Only searching once (sorting cost)
